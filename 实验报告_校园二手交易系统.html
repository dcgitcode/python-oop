🎯 场景

学校二手群里每天都有人发闲置物品信息，但是内容格式乱七八糟——有人不标价格、有人忘了写联系方式。 现在需要设计一个 Product 类，作为「商品档案卡」的模板。

📋 你需要完成的代码

  • 类名：Product
  • 属性（__init__ 里初始化）：
    • name —— 商品名（字符串），比如 "二手Python课本"
    • price —— 价格（数字），如果 ≤ 0，默认设为 0.01，不报错
    • seller —— 卖家姓名（字符串），比如 "你的名字"
    • status —— 状态，默认值为 "在售"
  • 方法1：__str__(self)
    → 返回字符串，格式为：【在售】商品名 | ¥价格 | 卖家：XXX

  • 方法2：is_expensive(self, threshold=500)
    → 如果 price ≥ threshold，返回 "高价商品"
    → 否则返回 "平价商品"
  • 方法3：mark_sold(self)
    → 把 status 改成 "已售"
    → 打印一行提示：商品 XXX 已标记为已售

🎯 场景

单个商品的档案卡做好了，但现在二手群里有几十件商品， 需要一个「管理系统」来批量管理它们。 请设计 ProductList 类。

📋 你需要完成的代码

  • 类名：ProductList
  • 属性：products —— 一个列表，存放 Product 对象
  • 方法1：add_product(self, product) 追加
    → 遍历 products，如果已存在 同名且同卖家 的商品，打印提示「已存在」并不添加
    → 否则追加到列表末尾
  • 方法2：remove_by_name(self, name) 删除
    → 按商品名删除，可能有多个同名但要全部删掉
    → 最后打印：已删除 N 件商品
  • 方法3：update_product(self, name, seller, new_price=None, new_status=None) 修改
    → 按照 商品名 + 卖家 两个条件定位商品（同名可能有不同卖家）
    → new_price 不为 None 时修改价格
    → new_status 不为 None 时修改状态
    → 找不到时打印：未找到该商品
  • 方法4：search_by_keyword(self, keyword) 查询
    → 按关键字搜索商品名（不区分大小写）
    → 返回符合条件的 Product 对象列表
