🔒 封装 Encapsulation

数据 + 方法打包在一起，外面不能乱改
🏦

银行卡 —— 你的余额别人改不了

你去 ATM 取钱：插卡 → 输密码 → 选金额 → 吐钞。
你不能直接把手伸进机器里改余额数字。

余额 = 私有属性，取款/存款 = 公开方法

💡 简单说：属性用 __xxx 保护起来，外部通过方法访问，这就是封装。

Python 示例 ↓

class Account: def __init__(self, owner, balance): self.owner = owner self.__balance = balance # 私有属性 def deposit(self, amount): # 公开方法 if amount > 0: self.__balance += amount def get_balance(self): # 公开方法 return self.__balance

❌ acc.__balance = 999999 → 外部改不了
✅ 只能通过 deposit() / get_balance()

🧬 继承 Inheritance

子类白拿父类的全部功能，还能再加自己的
🏫

学生和老师 —— 都需要登记"姓名+电话"

学校系统里，学生和老师都要登记姓名和电话。
如果两个类各写一遍，代码就重复了。

解法：Person 父类写一遍 → Student 和 Teacher 继承它，各自再加独有属性。

💡 继承 = 代码复用。父类写一遍，多个子类白拿。

Python 示例 ↓

class Person: # 父类 def __init__(self, name, phone): self.name = name self.phone = phone class Student(Person): # 继承 Person def __init__(self, name, phone, grade): super().__init__(name, phone) self.grade = grade # 新属性 class Teacher(Person): # 继承 Person def __init__(self, name, phone, title): super().__init__(name, phone) self.title = title # 新属性

✅ class 子类(父类): → 继承语法
✅ super().__init__() → 调用父类构造
✅ 子类可自由新增属性和方法

5/7

🎭 多态 Polymorphism

同一个方法名，不同对象各做各的事
💳

微信支付 vs 支付宝 —— 都叫"支付"

扫码付款时，收银台只调用 pay() 方法。
不管你用微信还是支付宝，各自走各自的流程。

调用方代码不用改——换个支付方式，只换对象，方法名不变。

💡 多态 = 同一个接口，不同实现。print() 能打印任何对象，因为每个类有自己的 __str__。

Python 示例 ↓

class WechatPay: def pay(self, amount): return f"微信支付 {amount} 元" class Alipay: def pay(self, amount): return f"支付宝支付 {amount} 元" # 同一个 checkout，不同支付结果！ def checkout(method, amount): print(method.pay(amount)) checkout(WechatPay(), 100) # 微信支付 100 元 checkout(Alipay(), 100) # 支付宝支付 100 元

✅ 方法名一样就能统一调用（鸭子类型）
✅ 不需要继承同一个父类
